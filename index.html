Buzuki

A mobile-first, responsive BEM flavoured flexbox css grid system.

Tags

There are 5 viewport sizes out-of-the-box, each with an associated @tag.

Modifiers with tags will only apply when the viewport reaches the size specified.

No @tag

Works across all viewport sizes

@xs

Minimum viewport size: 0px

@sm

Minimum viewport size: 544px

@md

Minimum viewport size: 768px

@lg

Minimum viewport size: 992px

@xl

Minimum viewport size: 1200px

The three stooges

Each component may be used independently or in unison, you decide.

Content

Defines a centred content area with a max width of 1200px out-of-the-box.

Row

Lays out it's child elements in a horizontal fashion, allowing you to do neat things.

Cell

Defines a region of content that abides by the rules, such as width, alignment, etc.

.Row

Defaults are equivalent to using .Row--flow-standard, .Row--valign-flex-start, .Row--gutters.

.Row--flow-standard
.Row--flow-reverse

flow-standard will lay child elements out from left to right, in DOM order. flow-reverse will lay child elements out from right to left, in reverse DOM order.

.Row--valign-baseline
.Row--valign-center
.Row--valign-flex-end
.Row--valign-flex-start
.Row--valign-stretch

Vertically aligns child elements based on the flexbox property "align-items" (baseline, center, flex-end, flex-start, stretch).

.Row--intersperse-pop
.Row--intersperse-push

intersperse-pop will remove the gutters in-between cells. intersperse-push will restore/add gutters in-between cells.

.Row--finish-short
.Row--finish-tall

finish-short will remove any space left at the bottom of the row from any gutters. finish-tall will restore/add the space left at the bottom of a row by any gutters.

.Row--no-gutters
.Row--gutters

no-gutters will remove all internal gutters. gutters will restore/add all internal gutters.

.Cell

Defines a horizontal grouping of child elements from left to right.

.Cell--hidden
.Cell--visible

hidden will hide the cell (display: none). visible will show the cell (display: block).

.Cell--has-rows

has-rows simply lets the cell know that you've embedded rows within it. This helps keeps things laid out how you'd expect it to look.

.Cell--#/#

#/# denotes any column size. If you're using 12 columns (set out-of-the-box), and you want a 12 column wide cell, you'd specify Cell--12/12. Need 6 a column wide cell? Cell--6/12. 2 column wide cell? Cell--2/12. You get the idea.

.Cell--offset-right-#/#
.Cell--offset-left-#/#

#/# denotes any column size. If you're using 12 columns (set out-of-the-box), and you want a 8 column wide offset, you'd specify Cell--8/12. Need 6 a column wide offset? Cell--6/12. 2 column wide offset? Cell--2/12. You get the idea.

.Cell--align-left
.Cell--align-center
.Cell--align-right
.Cell--align-justify

Horizontally aligns child elements based on the css property "text-align" (left, center, right, justify).

Customisation

Anything you see marked in yellow on this page can be modified.

When importing Buzuki via Sass, you may override any of the settings depending on your project needs.

More or less columns? Different breakpoints? Different content width? No problem.
